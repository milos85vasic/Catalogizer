üèóÔ∏è System Overview

Catalogizer is a comprehensive media collection management system that automatically detects, categorizes, and organizes your media files across SMB network sources. It provides real-time monitoring, advanced analytics, and multiple client interfaces for managing your entire media library.

üéØ Core Features

  • Automated media detection (50+ types)
  • Real-time SMB monitoring
  • SMB resilience with offline caching
  • Advanced analytics & quality analysis
  • JWT-based secure authentication
  • External metadata integration

  • catalog-api: Go REST API server
  • catalog-web: React web interface
  • installer-wizard: Desktop setup tool
  • catalogizer-desktop: Cross-platform desktop app
  • catalogizer-android: Mobile Android app
  • catalogizer-androidtv: Android TV app
  • catalogizer-api-client: TypeScript API client

‚öôÔ∏è Backend API Setup

The Go-based REST API server handles media detection, SMB monitoring, and provides the core functionality.

Manual Setup

1 Navigate to API Directory
cd catalog-api
2 Install Go Dependencies
go mod tidy
3 Configure Environment
cp .env.example .env # Edit .env with your settings

Key Configuration:

# Database DB_PATH=./data/catalogizer.db DB_ENCRYPTION_KEY=your-32-character-encryption-key # Server PORT=8080 HOST=127.0.0.1 GIN_MODE=debug # JWT Authentication JWT_SECRET=your-jwt-secret-key JWT_EXPIRY_HOURS=24 # SMB Sources SMB_SOURCES=smb://server1/media,smb://server2/videos SMB_USERNAME=your-username SMB_PASSWORD=your-password
4 Initialize Database
go run cmd/migrate/main.go
5 Start the API Server
go run cmd/server/main.go

Production Setup

1 Build Production Binary
CGO_ENABLED=1 go build -o catalogizer cmd/server/main.go
2 Create System Service
sudo cp catalogizer /usr/local/bin/ sudo useradd -r -s /bin/false catalogizer sudo mkdir -p /var/lib/catalogizer sudo chown catalogizer:catalogizer /var/lib/catalogizer
3 Create systemd Service

Create /etc/systemd/system/catalogizer-api.service:

[Unit] Description=Catalogizer API Server After=network.target [Service] Type=simple User=catalogizer Group=catalogizer WorkingDirectory=/var/lib/catalogizer ExecStart=/usr/local/bin/catalogizer Restart=always RestartSec=5 [Install] WantedBy=multi-user.target
4 Enable and Start Service
sudo systemctl enable catalogizer-api sudo systemctl start catalogizer-api sudo systemctl status catalogizer-api
üîç Verification: Check API health at http://localhost:8080/health

üîß Troubleshooting

Common issues and their solutions.

SMB Connection Issues

Symptoms: Media not detected, connection errors in logs
# Test SMB connectivity smbclient -L //your-server -U your-username # Check network ports nc -zv your-server 445 # Verify credentials SMB debugging logs

Database Problems

Symptoms: "Database locked" errors, slow queries
# Check database integrity sqlite3 catalogizer.db "PRAGMA integrity_check;" # Vacuum database sqlite3 catalogizer.db "VACUUM;" # Check permissions ls -la /var/lib/catalogizer/

Authentication Issues

Symptoms: Login failures, token errors
# Reset admin password go run cmd/admin/main.go reset-password admin newpassword # Check JWT configuration echo $JWT_SECRET | wc -c # Should be > 32 characters

Performance Issues

Symptoms: Slow UI, high CPU usage
# Monitor resources top -p $(pgrep catalogizer) # Check database performance EXPLAIN QUERY PLAN SELECT * FROM media_items; # Enable debug logging export LOG_LEVEL=debug

Web Interface Issues

Symptoms: White screen, API errors
# Check browser console # Verify API connectivity curl http://localhost:8080/health # Clear browser cache # Check CORS headers
